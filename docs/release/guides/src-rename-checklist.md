## 5. docs

Replace `src/` with `src/` in path examples and references in:

- `docs/release/pre-release-checklist.md`, `release-readiness-test-plan.md`, `release-readiness-summary.md`
- `docs/features/*.md` (file paths like `src/cuepoint/...`)
- `docs/future-features/prerelease/designs/03-quality-and-testing-depth.md`, `10-documentation-and-developer-experience.md`

## 6. Build artifacts

- `build/pyinstaller/*.toc` and similar are generated by PyInstaller; they will pick up the new `src` paths on the next build. No need to edit them by hand.
- Any build script or spec that explicitly uses the path `SRC` (e.g. pathex, datas) should be updated to `src`.

## 7. Actually renaming the directory

- On a **case-sensitive** filesystem (e.g. many Linux setups):  
  `git mv SRC src`
- On a **case-insensitive** filesystem (e.g. default macOS/Windows):  
  `git mv SRC src_temp && git mv src_temp src`  
  so the name change is detected by Git.

After renaming and updating the references above, run tests and a build to confirm:

- `pytest src/tests -v` (or `pytest` if `pytest.ini` is updated)
- `python main.py --help` (root wrapper)
- CI workflows
