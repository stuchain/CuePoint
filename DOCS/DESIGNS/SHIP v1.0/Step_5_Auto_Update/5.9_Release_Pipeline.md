# Implementation Step 5.9: Release Pipeline Integration

## Implementation Overview
**What We're Building**: A comprehensive release pipeline integration system that automates appcast generation, feed updates, release publishing, and all aspects of integrating the auto-update system into the CI/CD pipeline. This document defines the complete automation for generating update feeds, publishing releases, updating appcasts, and ensuring the update system works seamlessly with the release process.

**Implementation Details**

**5.9.1.1 Release Workflow Architecture**
- **Trigger**: GitHub Release creation (tag push)
- **Workflow File**: `.github/workflows/release.yml`
- **Workflow Steps**:
  1. **Build Artifacts** (if not already built):
     - Build macOS DMG
     - Build Windows installer
     - Sign artifacts
     - Notarize (macOS)
  2. **Generate Appcasts**:
     - Generate macOS appcast
     - Generate Windows appcast
     - Generate for stable channel
     - Generate for beta channel (if applicable)
  3. **Validate Feeds**:
     - Validate appcast XML structure
     - Validate all required fields
     - Validate URLs and signatures
     - Fail workflow if validation fails
  4. **Publish Feeds**:
     - Commit appcasts to `gh-pages` branch
     - Push to GitHub Pages
     - Verify feeds are accessible
  5. **Publish Release**:
     - Upload artifacts to GitHub Release
     - Add release notes
     - Tag release
- **Workflow Dependencies**:
  - Step 2: Build System (provides build infrastructure)
  - Step 3: macOS Packaging (provides DMG and signing)
  - Step 4: Windows Packaging (provides installer and signing)

**5.9.3.1 Feed Update Process**

  1. **Read Existing Feed**:
     - Read current appcast XML
     - Parse existing items
  2. **Add New Item**:
     - Create new item for release

  3. **Preserve History**:
     - Keep existing items
     - Limit to last 10-20 releases
  4. **Update Metadata**:
     - Update channel metadata if needed
     - Update feed description if needed
  5. **Write Feed**:
     - Write updated appcast XML
     - Commit to repository
- **Feed Ordering**: Latest version first (Sparkle/WinSparkle standard)

  - Enable rollback

- **History Cleanup**:
  - **Automatic**: Remove very old releases (optional)
  - **Manual**: Manual cleanup if feed gets too large
  - **Archive**: Archive old releases to separate feed (optional)

**5.9.3.3 Feed Rollback Procedures**

  3. **Update Feed**: Update appcast XML
  4. **Publish**: Commit and push updated feed
  5. **Verification**: Verify feed is updated
- **Rollback Scenarios**:
  - **Broken Release**: Remove broken release from feed

**5.9.7.1 Complete Release Workflow Example**
```yaml
name: Release and Update Feeds

on:
  release:
    types: [created]

jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.11'
      
      - name: Extract Version
        id: version
        run: |
          VERSION=${GITHUB_REF#refs/tags/v}
          echo "version=$VERSION" >> $GITHUB_OUTPUT
      
      - name: Generate macOS Appcast
        run: |
          python scripts/generate_appcast.py \
            --dmg dist/CuePoint-v${{ steps.version.outputs.version }}-macos-universal.dmg \
            --version ${{ steps.version.outputs.version }} \
            --url https://github.com/${{ github.repository }}/releases/download/${{ github.ref_name }}/CuePoint-v${{ steps.version.outputs.version }}-macos-universal.dmg \
            --notes https://github.com/${{ github.repository }}/releases/tag/${{ github.ref_name }} \
            --signature ${{ secrets.EDDSA_SIGNATURE }} \
            --output updates/macos/stable/appcast.xml
      
      - name: Generate Windows Appcast
        run: |
          python scripts/generate_update_feed.py \
            --exe dist/CuePoint-Setup-v${{ steps.version.outputs.version }}.exe \
            --version ${{ steps.version.outputs.version }} \
            --url https://github.com/${{ github.repository }}/releases/download/${{ github.ref_name }}/CuePoint-Setup-v${{ steps.version.outputs.version }}.exe \
            --notes https://github.com/${{ github.repository }}/releases/tag/${{ github.ref_name }} \
            --output updates/windows/stable/appcast.xml
      
      - name: Validate Feeds
        run: |
          python scripts/validate_feeds.py \
            --appcasts updates/macos/stable/appcast.xml updates/windows/stable/appcast.xml
      
      - name: Publish Feeds
        run: |
          python scripts/publish_feeds.py \
            --appcasts updates/macos/stable/appcast.xml updates/windows/stable/appcast.xml \
            --branch gh-pages \
            --message "Update feeds for ${{ github.ref_name }}"
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

**5.9.7.2 Feed Publishing Script Example**
```python
#!/usr/bin/env python3
"""Publish update feeds to GitHub Pages"""

import argparse
import subprocess
import sys
from pathlib import Path

def publish_feeds(appcast_paths, branch="gh-pages", message="Update feeds"):
    """Publish appcast feeds to GitHub Pages"""
    try:
        # Checkout gh-pages branch
        subprocess.run(["git", "checkout", branch], check=True)
        
        # Copy appcast files
        for appcast_path in appcast_paths:
            src = Path(appcast_path)
            dst = Path("updates") / src.parent.name / src.name
            dst.parent.mkdir(parents=True, exist_ok=True)
            dst.write_text(src.read_text())
        
        # Commit and push
        subprocess.run(["git", "add", "updates/"], check=True)
        subprocess.run(["git", "commit", "-m", message], check=True)
        subprocess.run(["git", "push", "origin", branch], check=True)
        
        print("Feeds published successfully")
        return True
    except subprocess.CalledProcessError as e:
        print(f"Error publishing feeds: {e}")
        return False

if __name__ == "__main__":
    parser = argparse.ArgumentParser(description="Publish update feeds")
    parser.add_argument("--appcasts", nargs="+", required=True, help="Appcast file paths")
    parser.add_argument("--branch", default="gh-pages", help="Git branch")
    parser.add_argument("--message", required=True, help="Commit message")
    args = parser.parse_args()
    
    success = publish_feeds(args.appcasts, args.branch, args.message)
    sys.exit(0 if success else 1)
```
